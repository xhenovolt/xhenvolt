# TiDB migration (Phase 2.7)

Database engine moved from Neon Postgres to TiDB Serverless (MySQL-compatible).

## Why

The "Neon down" symptom in the previous session was actually `DATABASE_URL`
unset — the local `.env.local` had already been swapped to `TIDB_*` vars
with `DATABASE_MODE=tidb`, but the codebase still expected the Postgres
DSN. Rather than re-introduce Postgres alongside the new credentials,
the engine was migrated to match the env.

## What changed

| Area | Before | After |
| --- | --- | --- |
| Driver | `@neondatabase/serverless` (HTTP) | `mysql2` (TCP+TLS) |
| Drizzle dialect | `drizzle-orm/neon-http` + `pg-core` | `drizzle-orm/mysql2` + `mysql-core` |
| ID column | `uuid PRIMARY KEY DEFAULT gen_random_uuid()` | `varchar(36) PRIMARY KEY` + app-side `randomUUID()` |
| JSON columns | `jsonb` | `json` |
| `timestamp with time zone` | yes | `timestamp(3)` (no tz; mysql2 round-trips Date) |
| Vector embeddings | `vector(1536)` via pgvector | `vector(1536)` via TiDB native VECTOR (wire format identical) |
| Returning inserted row | `.returning({...})` | pre-generate `id`, return that |
| Migration runner | `scripts/neon-http.ts` raw client | `scripts/_tidb.ts` mysql2 pool |
| Edge middleware | called `verifySession()` directly | cookie-presence check only; `(authed)/layout.tsx` does the real DB session lookup |

## Files touched

- `package.json` — drop `pg`, `@types/pg`, `@neondatabase/serverless`, `ws`, `@types/ws`. Add `mysql2`.
- `drizzle.config.ts` — `dialect: "mysql"`, URL assembled from `TIDB_*`.
- `src/lib/db/config.ts` — new. Reads TIDB_* / DB_* / MYSQL_* with fallback chain. Single source of truth for connection options.
- `src/lib/db/index.ts` — mysql2 pool + drizzle MySQL client. Same `hasDb()` contract.
- `src/lib/db/schema/*.ts` — all 18 files rewritten.
- `src/lib/env.ts` — ENV_VARS now lists TIDB_* as required.
- `src/lib/health/db-probe.ts` — opens a raw mysql2 connection, classifies failure into named causes (`auth_failed`, `dns_failed`, `tls_failed`, `unknown_database`, `host_refused`, `network_unreachable`, `timeout`, ...).
- `src/lib/auth/cookie.ts` — new. Holds `SESSION_COOKIE` constant for Edge middleware import without pulling in mysql2.
- `src/lib/auth/session.ts` — re-exports cookie constants; `purgeExpiredSessions` no longer uses `.returning()`.
- `src/lib/auth/users.ts` — `createAdmin` and `setAdminPassword` use `randomUUID()` + `affectedRows`.
- `middleware.ts` — Edge-safe: cookie presence only.
- `src/app/admin/(authed)/system-health/page.tsx` — TiDB cause labels and parsed-config card.
- `src/app/admin/(authed)/layout.tsx` — DB-down panel references TIDB_*.
- `src/app/admin/(authed)/testimonials/actions.ts`, `src/app/api/contact/route.ts` — pre-generated UUIDs in place of `.returning()`.
- `scripts/_tidb.ts` — new. Shared mysql2 pool factory for all scripts.
- `scripts/migrate.ts` — uses `__migrations` table on TiDB, applies `./drizzle/*.sql`.
- `scripts/create-admin.ts` — mysql2 + explicit `id`.
- `scripts/seed.ts` — stubbed; admin UI is the authoring path.
- `scripts/verify*.ts` — rewritten for mysql2.
- `drizzle/` — old Postgres SQL migrations deleted; will be regenerated by `npm run db:generate`.

## Required `.env.local`

```
TIDB_HOST=gateway01.<region>.prod.aws.tidbcloud.com
TIDB_PORT=4000
TIDB_USER=<user>
TIDB_PASSWORD=<password>
TIDB_DB=<database>
DATABASE_MODE=tidb
```

`DATABASE_URL` is no longer read. `DB_*` and `MYSQL_*` are accepted as fallbacks.

## One-time bring-up

```
npm install
npm run db:generate          # produces ./drizzle/*.sql in MySQL syntax
npm run db:migrate           # applies them to TiDB
npm run db:create-admin -- <email> <password> "<display name>"
npm run dev
```

Then visit `/admin/login`. If anything fails, `/admin/system-health` (or
`/api/admin/health/db`) names the cause.

## Risks and follow-ups

- **No automated seed.** Public-site content starts empty; author through `/admin`.
- **Vector retrieval not exercised yet.** Schema accepts `VECTOR(1536)` but no ingestion pipeline is shipped. When added, similarity must use TiDB's `VEC_COSINE_DISTANCE(a, b)`, not pgvector's `<=>`.
- **Text columns that used to be indexed are now `varchar(500)`** (MySQL can't index `TEXT` without a prefix length). If a URL legitimately exceeds 500 chars, the insert will fail loud.
- **`drizzle-kit push` is preferred over `migrate` during dev** because TiDB's HTAP layer is sensitive to long-running DDL. For production cutovers, prefer the generated SQL.
